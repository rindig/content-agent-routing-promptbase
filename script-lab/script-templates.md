# Script Templates and Frameworks

Templates, pre-scripting questions, and non-negotiable rules for all Eduba content formats. Read `hook-system.md` first for hook types.

---

## Before Scripting Any Video, Ask These Questions

Answer all seven before writing. Do not skip.

### 1. Which pillar does this map to? (P1-P5)

Must map to one pillar. If it doesn't fit cleanly, sharpen the angle or reconsider. See `../brand-vault/content-pillars.md` for full pillar definitions.

### 2. Where in the funnel does this sit? (top / middle / bottom)

Top = broad reach, Middle = assumes some context, Bottom = deep + connected to offer. See `../platform-playbook/funnel-map.md` for full tier definitions.

### 3. What hook type am I using? (1-6)

Refer to `hook-system.md` for the full breakdown. Choose deliberately. The hook type should match the content type and the insight being delivered.

### 4. What's the broad entry point and the niche depth?

Every piece of content needs both. The entry point is the thing that makes someone stop scrolling. The niche depth is the thing that makes someone follow. Define both before writing.

Example from the Hello World video:
- Broad entry point: "Everyone has written print Hello World"
- Niche depth: "Here are the 12,000 lines of code that execute when you do"

### 5. What offer does this implicitly connect to? (even if there's no CTA)

Every video should tie to an offer, even if the connection is never stated explicitly. This ensures that all content is quietly building toward business outcomes. See the Implicit Offer Architecture section below for the three patterns.

### 6. Can I explain this with a visual that builds progressively? (plays to animation strength)

This is a litmus test for the animated format. If the concept can be shown as layers being added, systems being built, or structures being revealed, it is a natural fit. If the concept is purely abstract with no visual analog, consider whether it is better suited for LinkedIn or needs a different angle.

### 7. Does the landing sentence reframe something? (signature move)

The last sentence of the video should shift the viewer's understanding. Not a summary. Not a recap. A reframe. The viewer should walk away seeing something differently than when they started.

## The Architecture of a Balanced Video

Every video functions at two levels:

1. **Broad Appeal Hook (first 5-10s)** — catches anyone in the target universe, not just experts
2. **Progressive Depth (remaining 20-50s)** — each sentence adds a layer of specificity. Some viewers drop off as depth increases; the ones who stay are the target audience

**Filter, Don't Exclude:** Start broad, narrow progressively. The hook is the wide end. The landing is the narrow end.

---

## The Implicit Offer Architecture

Every video ties to an offer implicitly. The audience feels taught, not sold to.

| Pattern | Volume | CTA | Approach |
|---------|--------|-----|----------|
| A: "Here's a problem, here's my approach" | 70% | None | Content IS the pitch |
| B: "Here's a method, it goes deeper" | 20% | Light | "Link in bio" after fully valuable video |
| C: "Here's what most get wrong" | 10% | None (implied) | Viewer builds mental model of Jake as solver |

Full offer connection strategy: `../platform-playbook/funnel-map.md`

## Short-Form Script Template (30-60 seconds, 80-120 words)

This is the primary content format. Animated explainers with voiceover. Every word counts. There is no room for filler, throat-clearing, or unnecessary qualifiers.

```
---
title: [Working title]
pillar: [P1/P2/P3/P4/P5]
hook-type: [quiet-reveal / number-anchor / reframe / counter-position / earned-authority / question-spiral]
funnel: [top / middle / bottom]
implicit-offer: [What product/service this naturally connects to]
status: [draft / review / final]
topic-id: [Reference to topic-engine, e.g. T-01]
---

HOOK (first 5-10 seconds):
[The opening line. Must land immediately. Use the hook type structure from hook-system.md.
This is the most important part of the script. If the hook doesn't work, nothing else matters.
Write multiple versions and pick the strongest.]

BUILD (middle 15-35 seconds):
[Progressive depth. Each sentence should add a new layer of understanding.
Think about what visual would accompany each beat -- the animation pipeline
will turn each beat into a visual layer.
Keep sentences short and declarative. No filler. No hedging.
Each sentence should earn its place.]

LAND (final 5-10 seconds):
[One sentence that reframes everything. Philosophical without being preachy.
The viewer should feel smarter after hearing this sentence.
This is the signature move -- the reframe that shifts understanding.]
```

### Writing Guidelines for 30-60 Second Scripts
- **Word count:** 80-120 words. This is not approximate. Going over creates pacing problems in the final video.
- **Sentence length:** Short. Averaging 8-12 words per sentence. This is voiceover, not prose.
- **Tone:** Calm, confident, conversational. Contractions are good. Jargon is fine if the audience knows it. Explain jargon if the audience might not.
- **One concept:** Teach exactly one thing. If you find yourself saying "and another thing," you have two videos, not one.
- **Visual thinking:** As you write each sentence in the BUILD section, imagine what appears on screen. If nothing visual comes to mind, the sentence might be too abstract.

## Short-Form Script Template (60-90 seconds, 120-180 words)

Same structural framework as the 30-60 second template, but with more room in the BUILD section for additional layers. Use this length when the concept requires one or two additional steps of explanation that cannot be compressed further.

```
---
title: [Working title]
pillar: [P1/P2/P3/P4/P5]
hook-type: [quiet-reveal / number-anchor / reframe / counter-position / earned-authority / question-spiral]
funnel: [top / middle / bottom]
implicit-offer: [What product/service this naturally connects to]
status: [draft / review / final]
topic-id: [Reference to topic-engine, e.g. T-01]
---

HOOK (first 5-10 seconds):
[Same requirements as 30-60 second template. The hook does not get longer just
because the video is longer. The hook is the hook.]

BUILD (middle 35-60 seconds):
[More room here for additional layers. Use this space for:
- A second example that reinforces the first
- A deeper technical explanation
- A counterpoint that strengthens the main argument
- An analogy that makes the concept click

Still no filler. Every sentence earns its place.
Continue thinking about visual beats -- each new idea is a new animation layer.]

LAND (final 5-10 seconds):
[Same requirement: one reframing sentence. The extra BUILD time means
the landing can reference more of the layers built during the video,
making the reframe more powerful.]
```

### When to Use 60-90 vs 30-60
- Use 30-60 when the concept is clean and lands in one move (most P5 counter-positions, simple P1 reveals)
- Use 60-90 when the concept needs a setup-and-payoff structure or when the visual build requires more steps (most P1 deep dives, complex P3 frameworks)
- When in doubt, try writing it at 30-60 first. If the concept feels cramped, expand to 60-90. Never start long and try to cut down.

## LinkedIn Post Template (150-300 words)

LinkedIn serves a different function than video. It can be more explicitly professional, more business-oriented, and more directly tied to offers. The audience on LinkedIn is closer to buyer personas -- decision-makers, executives, team leads -- than the broader video audience.

```
---
title: [Working title]
pillar: [P1/P2/P3/P4/P5]
funnel: [top / middle / bottom]
paired-video: [Reference to a short-form script if this accompanies one, otherwise "standalone"]
status: [draft / review / final]
---

HOOK (first 2-3 lines -- this is what shows before "see more"):
[Must be strong enough to earn the click. Same hook principles as video.
On LinkedIn, you have approximately 2-3 lines visible before the "see more" fold.
If these lines don't compel the reader to expand, the post is dead.
Use the same hook types from hook-system.md adapted for text.]

BODY (main content):
[This is where LinkedIn diverges from video. The body can take several forms:

Condensed video thesis + clip: Summarize the core insight from a video,
  add professional context, embed the video.

Industry counter-take: A more detailed version of a Counter-Position hook,
  with room for nuance and evidence.

"Here's how I approached X" story: Walk through a real project, decision,
  or client engagement. Professional storytelling.

Thought leadership piece: A framework, model, or perspective that
  demonstrates expertise. The 60/30/10 framework is a good example.

Regardless of type, the body should be:
- Scannable (short paragraphs, line breaks between ideas)
- Concrete (specific examples, real numbers, actual experiences)
- Valuable standalone (the reader should gain something even if they never click a link)]

CLOSE:
[Optional CTA. Keep it simple if present.
"I wrote a deeper breakdown of this. Link in comments."
"If you're navigating this at your org, I'd be curious what you're seeing."
No hard sells. No "DM me to learn more." The post earns interest; the CTA channels it.]
```

### LinkedIn-Specific Notes
- **Formatting matters.** Use line breaks generously. Dense paragraphs die on LinkedIn. Each idea gets its own line or short paragraph.
- **First line is everything.** The "see more" fold is your hook. Treat it with the same intensity as a video hook.
- **Engagement bait is off-brand.** No "Agree?" at the end. No "Like if you..." No "Comment your thoughts below." These tactics are transparent and erode trust.
- **Hashtags:** Use 3-5 relevant hashtags. Place them at the end, not inline. They are functional (discovery), not decorative.
- **Paired posts:** When a LinkedIn post accompanies a video, the post should add context the video doesn't have -- professional framing, business implications, a different angle on the same insight. Never just transcribe the video script.

## Long-Form Script Template (5-15 minutes, future state)

This format is not yet in production, but the template exists so the system is ready when it activates. Long-form content will likely expand on themes proven in short-form -- taking a concept that resonated in 60 seconds and giving it the full treatment.

```
---
title: [Working title]
pillar: [P1/P2/P3/P4/P5]
funnel: [middle / bottom]
based-on-shorts: [List of short-form scripts this expands on, e.g. "P1-hello-world-final.md, P2-60-30-10-framework-final.md"]
status: [draft / review / final]
---

COLD OPEN (first 30-60 seconds):
[Hook that establishes what the viewer will understand by the end.
Same hook types from hook-system.md, but with more room to breathe.
The cold open can be a full story, a detailed question spiral, or a
multi-layered quiet reveal. It should make the viewer commit to the full video.]

SECTION 1: [Section title]
[Each section teaches one clear concept. Think in terms of animation beats --
what builds on screen as this section progresses?
End each section with a transition that connects to the next.
The viewer should feel a sense of progression, not a list of unrelated ideas.]

SECTION 2: [Section title]
[Build on the previous section. Progressive complexity.
The viewer who understood Section 1 should find Section 2 accessible.
The viewer who skipped Section 1 should feel slightly lost -- that is the
progressive depth working as designed.]

[Additional sections as needed -- typically 3-5 sections for a 5-15 minute video]

SYNTHESIS:
[Bring all sections together. This is the "aha" moment where the pieces connect.
The viewer should feel the distinct concepts from each section clicking into
a unified understanding. This is where the real value lives -- not in any
individual section but in how they relate to each other.]

LANDING:
[The philosophical reframe. What has the viewer's understanding shifted to?
This is the same signature move as short-form but with more weight behind it.
After 5-15 minutes of progressive depth, the landing can make a bigger claim
because it has been earned through demonstration, not assertion.]
```

### Long-Form Structural Notes
- **Section count:** Aim for 3-5 sections. Fewer than 3 and the video likely doesn't need to be long-form. More than 5 and it likely needs to be a series.
- **Section length:** Each section should be 1-3 minutes. Short enough to maintain pacing, long enough to teach one concept properly.
- **Transitions:** Every section should end with a line that connects to the next section. "But that raises a question..." or "And this is where it gets interesting..." These transitions maintain momentum.
- **Based-on-shorts field:** Long-form content should expand on proven short-form topics whenever possible. This de-risks the format (the concept is already validated) and creates a content flywheel (short-form drives traffic to long-form).

## Non-Negotiable Rules

Every script, every format, every platform.

1. **Never hype.** No superlatives, no "INSANE" / "GAME-CHANGER" / "You NEED to see this."
2. **Never lead with credentials.** Lead with insight. Exception: Hook Type 5, sparingly.
3. **Make the viewer feel smarter, never dumber.** Core promise of all Eduba content.
4. **Understate.** Let facts do the heavy lifting. Adding emphasis weakens specificity.
5. **One concept per video.** If you can't state it in one sentence, the script isn't focused.
6. **Best CTA is no CTA.** Let value create demand.
7. **When you do CTA, keep it simple.** "Link in bio" or "I go deeper on this in [resource]." No scarcity tactics.

Full voice and tone rules: `../brand-vault/voice-and-tone.md`

## Script Review Checklist

Before marking any script as `review` or `final`, run through this checklist:

- [ ] Does the hook land in the first 5-10 seconds (video) or 2-3 lines (LinkedIn)?
- [ ] Is the hook type used deliberately and correctly?
- [ ] Does the script teach exactly one concept?
- [ ] Is the word count within range for the target format?
- [ ] Does every sentence earn its place? (Read each sentence and ask: if I removed this, would the script be worse?)
- [ ] Does the landing sentence reframe something?
- [ ] Does the script sound like Jake? (Calm, precise, understated, conversational)
- [ ] Is there any hype language? (If yes, remove it)
- [ ] Are credentials used only when they serve the insight?
- [ ] Would the viewer feel smarter after consuming this?
- [ ] Can each BUILD beat be visualized as an animation layer?
- [ ] Is the frontmatter complete? (pillar, hook-type, funnel, implicit-offer, status, topic-id)
- [ ] Does the filename follow the naming convention?
