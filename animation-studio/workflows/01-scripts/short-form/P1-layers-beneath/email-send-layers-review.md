## SCRIPT

**HOOK (0-5s):**
When you hit "Send" on an email, it passes through at least five different servers before reaching the recipient.

**BUILD (5-30s):**
Here's the journey. Your email client sends it to your SMTP server using one protocol. That server looks up the recipient's mail server using DNS. It opens a connection to their server using another protocol. Their server checks if the recipient exists, runs spam filters, virus scans, and authenticates the sender.

If everything checks out, it stores the email. When the recipient opens their inbox, their client connects to their mail server, downloads the message, and displays it.

Five servers. Three different protocols. Multiple security checks. All of it happening in seconds.

**LAND (30-35s):**
Email feels instant and simple. Underneath, it's a conversation between servers spanning the globe, each one doing its job so you never have to think about it.
